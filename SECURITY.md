# SECURITY.md

## Política de Segurança

A segurança deste projeto é levada a sério.

Se você identificar uma vulnerabilidade, falha de segurança, comportamento inesperado ou risco potencial, reporte de forma responsável.

### Versões Suportadas

| Versão | Suporte |
| ------ | ------- |
| 1.x    | ✅       |
| 0.x    | ❌       |

### Reportando Vulnerabilidades

Envie informações contendo:

* Descrição da vulnerabilidade
* Passos para reprodução
* Impacto esperado
* Evidências técnicas (opcional)

### Divulgação Responsável

Solicitamos que vulnerabilidades não sejam divulgadas publicamente antes da correção oficial.

### Escopo

Esta política cobre:

* HTML
* CSS
* JavaScript
* Assets
* GitHub Pages
* Workflows GitHub Actions

### Boas Práticas Recomendadas

* Utilizar HTTPS
* Ativar autenticação em dois fatores (2FA)
* Revisar Pull Requests antes do merge
* Evitar exposição de chaves privadas
* Não armazenar credenciais no repositório

### Isenção

Este projeto é fornecido "como está", sem garantias de qualquer natureza.
